Analisis Klasifikasi Kesuburan Tanah#

Metode: K-Nearest Neighbors (KNN)

1. Pemrosesan Data (Preprocessing)#

Tahapan ini dilakukan untuk memastikan data dalam kondisi siap pakai sebelum masuk ke model klasifikasi:

  • Handling Missing Values: Dataset memiliki nilai yang hilang (misal: 12% pada P Tersedia). Data kosong pada fitur numerik diisi dengan nilai Median, sedangkan fitur kategorikal (Tekstur Tanah) diisi dengan Modus.

  • Encoding (One-to-Many): Fitur “Tekstur Tanah” diubah dari teks menjadi kolom angka (0 dan 1). Tahap ini wajib karena algoritme KNN bekerja berdasarkan perhitungan jarak matematis.

  • Normalisasi (Min-Max): Seluruh fitur numerik diseragamkan skalanya ke rentang 0.0 - 1.0. Ini dilakukan agar fitur dengan angka besar (seperti Kejenuhan Basa) tidak mendominasi fitur berangka kecil (seperti pH).

  • Partitioning (Pembagian Data): Dataset dibagi menjadi dua bagian:

    • Data Latih (80%): 1.600 sampel digunakan untuk melatih model.

    • Data Uji (20%): 400 sampel digunakan untuk menguji akurasi model.

2. Analisis K-Nearest Neighbors (KNN)#

Model dikonfigurasi dengan parameter sebagai berikut:

  • Jumlah Tetangga (K): 5 (Angka ganjil untuk menghindari hasil seri dalam klasifikasi).

  • Fungsi Jarak: Euclidean Distance (Mencari kemiripan berdasarkan jarak terpendek antar titik data).

KESIMPULAN#

Berdasarkan analisis di atas, model KNN mampu melakukan klasifikasi kesuburan tanah dengan tingkat keberhasilan 80,25%.

Meskipun terdapat banyak missing values pada dataset awal, langkah preprocessing (terutama normalisasi dan imputasi) terbukti sangat krusial dalam menjaga stabilitas model. Model ini memiliki nilai Precision (0,821) yang lebih tinggi daripada Recall, yang berarti model cenderung sangat berhati-hati dan akurat dalam menentukan bahwa suatu lahan benar-benar berkategorikal pada Tanah yang Subur.
